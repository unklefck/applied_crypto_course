### **Прикладная криптография — Финальный проект за 1 семестр:**
**«CryptoSafe Manager»** – Кроссплатформенный менеджер паролей с графическим интерфейсом, зашифрованной локальной базой данных и защищенной работой с буфером обмена.

---

### **Стек технологий:**
- **Язык:** Python (наиболее доступный, отличные криптобиблиотеки)
- **Фреймворк для GUI:** Tkinter (встроенный) или PyQt (более профессиональный)
- **База данных:** SQLite (с шифрованием "на покое" через SQLCipher или шифрование на уровне приложения)
- **Криптобиблиотеки:** `cryptography`, `pycryptodome`, `argon2-cffi`
- **Защита буфера обмена:** Собственный защищенный буфер обмена с автоочисткой

---

### **Разбивка проекта на этапы (8 спринтов)**

#### **Спринт 1: Основа — Защищенная база данных и каркас GUI**
- **Цель:** Создать зашифрованную базу данных SQLite и основное окно графического интерфейса.
- **Результаты:**
    - Схема SQLite для хранилища (записи, журнал аудита, настройки).
    - Шифрование конфиденциальных полей на уровне приложения перед вставкой в БД.
    - Главное окно с меню, списком записей и заглушками для кнопок.
- **Ключевая функция:** Первоначальная настройка с созданием мастер-пароля.

#### **Спринт 2: Мастер-пароль и управление ключами**
- **Цель:** Реализовать Argon2 KDF и безопасное формирование ключей.
- **Результаты:**
    - Хранение хэша Argon2 от мастер-пароля (для проверки).
    - Формирование ключа шифрования (AES-256) из мастер-пароля + соли.
    - Защищенное кэширование ключа в памяти (опционально с интеграцией в keychain/ОС).
- **Ключевая функция:** Логика смены пароля и ротации ключей.

#### **Спринт 3: Основные операции с хранилищем (CRUD)**
- **Цель:** Добавление, редактирование, удаление и отображение учетных данных с безопасным шифрованием.
- **Результаты:**
    - Использование AES-256-GCM для шифрования каждой записи (уникальный одноразовый номер каждый раз).
    - Отображение записей в таблице GUI (пароль скрыт).
    - Функционал поиска/фильтрации.
- **Ключевая функция:** Автогенерация надежных паролей (с использованием CSPRNG).

#### **Спринт 4: Защищенный буфер обмена с автоочисткой**
- **Цель:** Реализовать копирование в буфер обмена с автоматической очисткой через время.
- **Результаты:**
    - Кнопка «Копировать пароль» для каждой записи.
    - Очистка буфера обмена через настраиваемое время (например, 30 секунд).
    - Уведомление в системном трее при очистке буфера.
    - **Безопасность:** Использование `pyperclip` или нативных API платформы; гарантия, что данные в буфере не хранятся в открытом виде в памяти дольше необходимого.
- **Ключевая функция:** Предпросмотр содержимого буфера (частично скрытое) и опция немедленной очистки.

#### **Спринт 5: Аудит-логи и защита целостности**
- **Цель:** Защищенные от подделки журналы аудита с цифровой подписью.
- **Результаты:**
    - Подпись каждой записи лога (действие, временная метка, пользователь) с помощью HMAC или Ed25519.
    - Отображение журнала аудита в GUI (только для просмотра, без редактирования).
    - Проверка целостности журнала при запуске.
- **Ключевая функция:** Экспорт журнала аудита в виде подписанного JSON для внешней проверки.

#### **Спринт 6: Импорт/Экспорт и безопасный обмен**
- **Цель:** Реализовать зашифрованный экспорт/импорт и безопасный обмен учетными данными.
- **Результаты:**
    - Экспорт хранилища в зашифрованный JSON (с использованием открытого RSA/ECC ключа получателя).
    - Импорт из распространенных форматов (CSV, JSON) с шифрованием во время импорта.
    - **Опционально:** Обмен через QR-код для публичных ключей.
- **Ключевая функция:** Обмен отдельными записями без раскрытия основного хранилища.

#### **Спринт 7: Усиление защиты и улучшение UX**
- **Цель:** Защита от атак по побочным каналам и улучшение удобства использования.
- **Результаты:**
    - Сравнение паролей за константное время.
    - Затирание конфиденциальных данных в памяти (с использованием `ctypes` или `secrets`).
    - Автоблокировка после бездействия.
    - Иконка в трее и сворачивание в трей.
- **Ключевая функция:** «Режим паники» – быстрое закрытие и очистка буфера обмена.

#### **Спринт 8: Финальная интеграция, тестирование и документация**
- **Цель:** Готовое к использованию приложение с полной документацией.
- **Результаты:**
    - Комплексные модульные тесты (например, `pytest`) для криптографических функций.
    - Скрипт сборки исполняемого файла (PyInstaller).
    - Руководство пользователя (PDF или встроенное) с объяснением функций безопасности.
    - Демонстрационное видео, показывающее:
        - Добавление учетных данных
        - Безопасное копирование/вставку
        - Просмотр журнала аудита
        - Процесс экспорта/импорта
- **Ключевая функция:** Восстановление после ошибок и механизм резервного копирования.

### **Дополнительные возможности (Бонус/Дополнительный балл)**
1. **Интеграция с браузером** – Автозаполнение через защищенный локальный сокет.
2. **Поддержка двухфакторной аутентификации** – Генерация TOTP кодов (RFC 6238).
3. **Синхронизация через облако** – Сквозное зашифрованное резервное копирование через API Dropbox/Google Drive.
4. **Разблокировка по биометрии** – Интеграция с Windows Hello / macOS Touch ID.
5. **Проверка на утечки паролей** – API HaveIBeenPwned (k-анонимность через SHA-1).

---

### **Акценты на аспектах безопасности:**
- **Шифрование "на покое":** Каждая запись шифруется индивидуально (AES-GCM).
- **Разделение ключей:** Разные ключи для шифрования и аутентификации.
- **Защищенная память:** Конфиденциальные данные обнуляются после использования.
- **Изоляция буфера обмена:** Использование приватного буфера обмена, где это возможно (Linux/macOS).
- **Отсутствие сетевого взаимодействия:** Полностью локальное приложение, если не добавлена бонусная облачная синхронизация.
