### **Проект: CryptoCore — документ технических требований (Спринт 3)**

**Цель спринта:** Реализовать безопасный источник случайности для ключей и IV.

#### **1. Структура проекта и чистота репозитория**

Нужно создать новый модуль CSPRNG и встроить его в существующую архитектуру.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| STR-1 | Все требования из предыдущих спринтов (STR-1 ... STR-4) **должны** по-прежнему выполняться.                       | Must      |
| STR-2 | **Должен** быть создан новый выделенный исходный файл для модуля CSPRNG.                                            | Must      |
|      | - **Рекомендуемый путь:** `src/csprng.py` (Python) или `src/csprng.c` / `src/csprng.h` (C)                        |           |
| STR-3 | Файл `README.md` **должен** быть обновлён и включать:                                                              | Must      |
|      | - Документацию по новому поведению, когда `--key` не указан.                                                        |           |
|      | - Пример, где инструмент генерирует и выводит случайный ключ.                                                       |           |
|      | - Примечание о свойствах безопасности используемого CSPRNG.                                                         |           |
|      | - Инструкции по запуску статистического тестового набора NIST.                                                      |           |

#### **2. Парсер интерфейса командной строки (CLI)**

CLI должен быть расширен для поддержки необязательной генерации ключа.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| CLI-1 | Аргумент `--key` **должен** стать необязательным для операций шифрования.                                          | Must      |
| CLI-2 | Если аргумент `--key` передан, инструмент **должен** использовать его точно так же, как в предыдущих спринтах.    | Must      |
| CLI-3 | Если аргумент `--key` **не** передан при **шифровании**:                                                            | Must      |
|      | - Инструмент **должен** сгенерировать безопасный случайный ключ длиной 16 байт (128 бит).                         |           |
|      | - Сгенерированный ключ **должен** быть выведен в стандартный вывод (`stdout`) как шестнадцатеричная строка с понятной меткой. |           |
|      | - После этого инструмент **должен** продолжить шифрование с использованием сгенерированного ключа.                |           |
| CLI-4 | Для операций **расшифрования** аргумент `--key` **должен** оставаться обязательным. Если он не передан, инструмент **должен** вывести ошибку и завершиться. | Must |
| CLI-5 | Если ключ, предоставленный пользователем, выглядит слабым (например, все нули или последовательные байты), инструменту **следует** вывести предупреждение в `stderr`. Это требование уровня "Should" для поощрения лучших практик. | Should |

**Примеры вызова:**
```bash
# Шифрование с автоматической генерацией ключа
$ cryptocore --algorithm aes --mode ctr --encrypt --input plaintext.txt --output ciphertext.bin
> [INFO] Generated random key: 1a2b3c4d5e6f7890fedcba9876543210

# Расшифрование (ключ всегда должен быть передан)
$ cryptocore --algorithm aes --mode ctr --decrypt --key 1a2b3c4d5e6f7890fedcba9876543210 --input ciphertext.bin --output decrypted.txt
```

#### **3. Криптографически стойкая генерация случайных чисел (CSPRNG)**

Сердцевина этого спринта — реализация безопасного выделенного модуля генерации случайных данных.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| RNG-1 | В новом модуле CSPRNG **должна** быть реализована выделенная функция (или набор функций).                         | Must      |
| RNG-2 | Основная функция **должна** иметь сигнатуру `generate_random_bytes(num_bytes: int) -> bytes` (Python) или эквивалент на C. | Must |
| RNG-3 | Реализация **должна** использовать криптографически стойкий источник случайности, предоставляемый ОС или проверенной библиотекой: | Must |
|      | - **Для Python:** **должна** использоваться функция `os.urandom()`. Модуль `secrets` также допустим, но `os.urandom` более прямолинеен. |           |
|      | - **Для C:** **должно** использоваться устройство `/dev/urandom` через открытие файла и чтение байтов, **либо** функция `RAND_bytes()` из OpenSSL. |           |
| RNG-4 | Модуль CSPRNG **не должен** использовать стандартные функции псевдослучайности (например, модуль `random` в Python или `rand()` в C), так как они не являются криптографически стойкими. | Must |
| RNG-5 | Функция `generate_random_bytes` **должна** быть встроена в основную логику инструмента для:                       | Must      |
|      | - Генерации ключа шифрования, когда `--key` не задан.                                                               |           |
|      | - Генерации всех IV для операций шифрования (в продолжение Спринта 2).                                             |           |
| RNG-6 | Функция **должна** корректно обрабатывать возможные ошибки (например, невозможность открыть `/dev/urandom`, сбой `os.urandom`) и выдавать понятное, полезное сообщение об ошибке или исключение. | Must |

**Примеры кода CSPRNG:**
```python
# Python: src/csprng.py
import os

def generate_random_bytes(num_bytes):
    """Генерирует криптографически стойкую случайную строку байтов."""
    return os.urandom(num_bytes)
```

```c
// C: src/csprng.c (с использованием OpenSSL)
#include <openssl/rand.h>

int generate_random_bytes(unsigned char *buffer, int num_bytes) {
    if (RAND_bytes(buffer, num_bytes) != 1) {
        // Обработка ошибки: RAND_bytes завершилась неуспешно
        return -1;
    }
    return 0;
}
```

#### **4. Управление ключами и IV**

Работа с криптографическим материалом должна быть обновлена с использованием нового CSPRNG.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| KEY-1 | При шифровании, если ключ генерируется автоматически, его длина **должна** составлять 16 байт для AES-128.      | Must      |
| KEY-2 | Сгенерированный ключ **должен** быть выведен в `stdout` в понятном шестнадцатеричном формате ровно один раз и сразу после генерации. | Must |
| KEY-3 | Инструмент **не должен** записывать сгенерированный ключ в выходной файл с шифртекстом. Пользователь сам отвечает за сохранение ключа, выведенного в терминал. | Must |
| IV-1 | Генерация IV для всех соответствующих режимов (CBC, CFB, OFB, CTR) **должна** использовать новую функцию `generate_random_bytes`. | Must |
| IV-2 | Обработка IV из Спринта 2 (добавление к шифртексту, чтение из файла/CLI при расшифровании) **должна** оставаться без изменений. | Must |

#### **5. Тестирование и верификация**

Теперь тестирование должно включать строгий статистический анализ с использованием набора тестов NIST.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| TEST-1 | **Тест генерации ключа:** запуск инструмента для шифрования без опции `--key` **должен** приводить к выводу ключа в терминал и успешному шифрованию. Последующее расшифрование с этим ключом **должно** восстанавливать исходный файл. | Must |
| TEST-2 | **Тест уникальности:** **должен** быть создан тестовый сценарий, который вызывает `generate_random_bytes` 1000 раз для генерации 1000 ключей по 16 байт. | Must |
|      | - Тест **должен** проверять, что все 1000 ключей уникальны (без дубликатов).                                       |           |
|      | - Вероятность дубликата в 1000 выборках для настоящего CSPRNG исчезающе мала, поэтому любой повтор указывает на критический дефект. |           |
| TEST-3 | **Статистический набор тестов NIST:** студент **должен** запустить NIST Statistical Test Suite (STS) на выходных данных своего CSPRNG. | Must |
|      | - **Тестовые данные:** сгенерировать достаточно большой бинарный файл (рекомендуется 1-100 МБ), заполненный выводом `generate_random_bytes`. |           |
|      | - **Инструмент:** использовать NIST STS (доступен на [сайте NIST](https://csrc.nist.gov/projects/random-bit-generation/documentation-and-software)). |           |
|      | - **Процедура:** следовать документации NIST STS и запустить все 15 тестов на сгенерированных данных.             |           |
|      | - **Критерий успеха:** большая часть тестов должна проходить (`p-value >= 0.01`). Небольшое число отказов статистически допустимо, но массовые провалы указывают на ошибочный генератор. |           |
|      | - **Документация:** процедура тестирования и результаты **должны** быть описаны в `README.md` или отдельном файле `TESTING.md`. |           |
| TEST-4 | **Базовый тест распределения:** тестовому сценарию **следует** выполнять простую проверку энтропии, например убеждаться, что у сгенерированных ключей средний вес Хэмминга высокий (около 50% битов равны `1`). | Should |
| TEST-5 | **Тест совместимости:** инструмент по-прежнему **должен** проходить все тесты совместимости из Спринта 2. В этих тестах будет использоваться аргумент `--key`, поэтому RNG в них не участвует. | Must |

**Пример процедуры тестирования NIST:**
```bash
# Пример шагов для набора тестов NIST (вариант на Python)
# 1. Установить NIST STS или скачать C-реализацию и собрать её

# 4. Проанализировать результаты в сформированных отчётах
```

**Пример тестового сценария на уникальность и базовую статистику (Python):**
```python
# tests/test_csprng.py
from src.csprng import generate_random_bytes

def test_key_uniqueness():
    key_set = set()
    num_keys = 1000
    for _ in range(num_keys):
        key = generate_random_bytes(16)
        key_hex = key.hex()
        # Проверка уникальности
        assert key_hex not in key_set, f"Duplicate key found: {key_hex}"
        key_set.add(key_hex)
    print(f"Successfully generated {len(key_set)} unique keys.")

def test_nist_preparation():
    """Генерация большого случайного файла для тестирования NIST"""
    total_size = 10_000_000  # 10 MB
    with open('nist_test_data.bin', 'wb') as f:
        bytes_written = 0
        while bytes_written < total_size:
            chunk_size = min(4096, total_size - bytes_written)
            random_chunk = generate_random_bytes(chunk_size)
            f.write(random_chunk)
            bytes_written += chunk_size
    print(f"Generated {bytes_written} bytes for NIST testing in 'nist_test_data.bin'")
```
