#### **2. Парсер интерфейса командной строки (CLI)**

Для операций хеширования должна быть реализована новая подкоманда, отдельная от шифрования и расшифрования.

| ID  | Описание требования                                                                                                 | Приоритет |
| :-- | :------------------------------------------------------------------------------------------------------------------ | :-------- |
| CLI-1 | Инструмент **должен** поддерживать новую подкоманду `dgst` для вычисления message digest.                          | Must      |
| CLI-2 | Подкоманда `dgst` **должна** принимать следующие аргументы:                                                         | Must      |
|      | - `--algorithm ALGORITHM` : **должна** поддерживаться как минимум `sha256` и ещё один алгоритм (например, `sha3-256`, `blake2`). |           |
|      | - `--input INPUT_FILE` : **должен** принимать путь к входному файлу, для которого нужно вычислить хеш.            |           |
| CLI-3 | Команда `dgst` **не должна** требовать или принимать аргументы, специфичные для шифрования (`--key`, `--mode`, `--encrypt/--decrypt`, `--iv`). | Must |
| CLI-4 | Вывод **должен** печататься в `stdout` в формате `HASH_VALUE  INPUT_FILE_PATH` (по образцу стандартных `*sum`-утилит). | Must |
| CLI-5 | Инструменту **следует** поддерживать необязательный флаг `--output FILE`, чтобы записывать хеш в файл вместо `stdout`. | Should |

**Примеры вызова:**
```bash
# Базовое вычисление хеша
$ cryptocore dgst --algorithm sha256 --input document.pdf
> 5d5b09f6dcb2d53a5fffc60c4ac0d55fb052072fa2fe5d95f011b5d5d5b0b0b5  document.pdf

# Вычисление хеша с записью в файл
$ cryptocore dgst --algorithm sha3-256 --input backup.tar --output backup.sha3
```

Этот спринт требует существенной криптографической реализации самих хеш-функций.

**Пример тестового сценария для лавинного эффекта:**
```python
# tests/test_hash_avalanche.py
from src.hash.sha256 import SHA256

def test_avalanche_effect():
    """Проверка того, что изменение одного бита порождает полностью иной хеш"""
    original_data = b"Hello, world!"
    modified_data = b"Hello, world?"  # Изменён последний символ

    sha256 = SHA256()
    hash1 = sha256.hash(original_data)
    sha256 = SHA256()  # Сброс состояния
    hash2 = sha256.hash(modified_data)

    # Перевести в бинарный вид и посчитать число отличающихся битов
    bin1 = bin(int(hash1, 16))[2:].zfill(256)
    bin2 = bin(int(hash2, 16))[2:].zfill(256)

    diff_count = sum(bit1 != bit2 for bit1, bit2 in zip(bin1, bin2))

    print(f"Bits changed: {diff_count}/256")
    # Лавинный эффект: должно изменяться примерно 128 бит (50%)
    assert 100 < diff_count < 156, f"Avalanche effect weak: only {diff_count} bits changed"
```
