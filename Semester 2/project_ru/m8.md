**Пример структуры запуска тестов:**
```python
# tests/run_tests.py
import unittest
import sys

def run_all_tests():
    # Найти и выполнить все тесты
    loader = unittest.TestLoader()
    suite = loader.discover('tests/unit', pattern='test_*.py')
    suite.addTests(loader.discover('tests/integration', pattern='test_*.py'))

    runner = unittest.TextTestRunner(verbosity=2)
    result = runner.run(suite)

    return result.wasSuccessful()

if __name__ == '__main__':
    success = run_all_tests()
    sys.exit(0 if success else 1)
```

**Пример формата документации API:**
````markdown
# Документация API CryptoCore

## Модуль: cryptocore.aes

### `encrypt_block(key, plaintext)`
Шифрует один 16-байтный блок с помощью AES-128.

**Параметры:**
- `key` (bytes): 16-байтный ключ шифрования
- `plaintext` (bytes): 16-байтный блок для шифрования

**Возвращает:**
- `bytes`: 16-байтный зашифрованный блок

**Вызывает:**
- `ValueError`: если длина ключа или открытого текста неверна

**Пример:**
```python
from cryptocore.aes import encrypt_block

key = b'0' * 16
plaintext = b'hello world!!!!!'
ciphertext = encrypt_block(key, plaintext)
```

**Замечания по безопасности:**
- Никогда не используйте режим ECB для нескольких блоков. Вместо этого применяйте режимы аутентифицированного шифрования, например GCM.
````

**Пример раздела руководства пользователя:**
````markdown
## Шифрование и расшифрование файлов

### Базовое шифрование AES

Чтобы зашифровать файл с помощью AES-256 в режиме CBC:
```bash
cryptocore --algorithm aes --mode cbc --encrypt \
  --key 00112233445566778899aabbccddeeff00112233445566778899aabbccddeeff \
  --input secret.txt \
  --output secret.enc
```

### Работа с GCM и associated data

GCM обеспечивает и конфиденциальность, и контроль целостности. Можно также передавать дополнительные аутентифицируемые данные (AAD):

```bash
# Шифрование с AAD
cryptocore --algorithm aes --mode gcm --encrypt \
  --key 00112233445566778899aabbccddeeff \
  --input database.sql \
  --output database.enc \
  --aad "database_version_3.2"

# Расшифрование с тем же AAD (проверка выполняется автоматически)
cryptocore --algorithm aes --mode gcm --decrypt \
  --key 00112233445566778899aabbccddeeff \
  --input database.enc \
  --output database_decrypted.sql \
  --aad "database_version_3.2"
```

**Важно:** если AAD не совпадает или шифртекст был изменён, инструмент завершится с ошибкой и не выведет данные.
````
